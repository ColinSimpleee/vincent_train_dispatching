# 列车调度模拟器 — UI 重构功能说明

> 面向 UI 重构的功能需求文档。本文档只描述"UI 要支持哪些功能、展示什么信息、响应什么交互"，不约束具体视觉风格、布局位置与配色，留给设计师自由发挥。

---

## 1. 项目是做什么的

一款面向 Web（Vue 3 + SVG）的实时铁路调度模拟器。玩家扮演车站调度员，在一个拓扑化的线路图上指挥多列动车组进站、停靠、发车，目标是：

- **安全**：避免列车物理碰撞（碰撞 = 游戏失败）
- **准点**：按时刻表接发列车，尽量减少晚点
- **吞吐**：在持续涌入的列车流中合理分配站台

核心玩法循环：时刻表不断生成新列车 → 列车进入等待队列 → 玩家下达"允许进站"指令 → 列车按规划路径进站 → 自动停站 30–60 秒上客 → 玩家下达"发车"指令 → 列车驶出控制边界、移交控制权 → 循环。

游戏以 60 tick/秒 的虚拟时间运行，支持 1×/2×/5×/10× 倍速与暂停。

---

## 2. UI 需要承担的功能模块

整个应用由**两个顶层视图**组成：

1. **开始界面（StartScreen）**：车站选择
2. **游戏界面（Game）**：实际调度场景

游戏界面又分为若干功能区块，以下按"功能"而非"位置"组织。

---

## 3. 开始界面（StartScreen）

### 3.1 功能
- 展示所有可选车站，让玩家选择一个进入。
- 明确标示哪些车站可玩、哪些为占位/未开放（禁用态）。

### 3.2 每个车站卡片需展示
| 信息 | 说明 |
|------|------|
| 车站名称 | 如"新手教学站""终端车站""中心枢纽站" |
| 难度等级 | 1–5 星级（`difficulty` 字段） |
| 车站类型标签 | `small` / `terminal` / `hub` |
| 简介描述 | 一句话说明特点（道岔复杂度、流量等） |
| 进入按钮 | 占位站需呈禁用态并不可点 |

### 3.3 交互
- 点击卡片的进入按钮 → 进入对应车站的游戏界面。
- 禁用卡片不响应点击。

---

## 4. 游戏界面 — 主线路图（GameCanvas）

**这是玩家最主要的注视区域。** 所有调度决策都围绕这张图进行。

### 4.1 需要渲染的静态元素

| 元素 | 功能意义 |
|------|----------|
| **轨道（edge）** | 每条边有起止节点、长度；可能是直线或贝塞尔曲线；需按"道床 + 钢轨"层次感呈现 |
| **站台区域** | 矩形区域，叠加在两条轨道之间，需带编号标签（如 "1 (Main)"） |
| **道岔节点（switch）** | 圆形控件，显示当前激活出边索引（如 S1/S2），玩家可点击切换 |
| **信号灯节点（signal）** | 带灯箱与灯杆的控件，显示红/绿两态，玩家可点击切换 |
| **车挡（buffer_stop）** | 终端轨道尽头的警示标志 |
| **控制边界标记** | 出口轨道上的"移交点"标记，越过后列车脱离玩家控制 |

### 4.2 道岔与信号的交互状态

- **道岔**：点击切换到下一条出边，画布需高亮当前激活的出边路径（让玩家一眼看出"扳向了哪里"）。
- **信号灯**：点击红↔绿切换。红灯阻止列车通过、阻止停站列车恢复行驶，状态必须一目了然。
- **hover 态**：鼠标悬停时给出可点击暗示。
- **激活反馈**：点击后应有即时视觉反馈。

### 4.3 列车渲染

每列车由 8 或 16 节车厢（取决于是否重联）沿路径连续排布，需支持：

- **车型区分**：3 种车型（CR400AF / CR400BF / CRH380A），需要通过色彩/条纹等视觉差异化区分。
- **车头朝向**：首尾车厢需有流线型外观；列车能沿贝塞尔曲线弯折，每节车厢有独立旋转角度。
- **车号标签**：漂浮在首节车厢上方（如 "G501"），始终可读。
- **车厢细节**：窗户、条纹等构成"能看出是一列动车"的视觉标志。

### 4.4 列车状态的画布提示

以下状态需要**在画布上**直接呈现，无需玩家切换面板即可感知：

- **停站中（BOARDING）**：停在站台上等待乘客，视觉上应与"运行中"区分。
- **可以出站（READY）**：车头旁边浮出一个"发车"按钮，点击即发车（与面板按钮等价的快捷入口）。
- **被选中**：玩家选定的列车在画布上应有高亮/选中边框。

### 4.5 键盘模式叠加层

进入键盘模式时，画布上叠加一层字母/数字标签：

- 每个道岔旁边显示一个数字键（1–9, 0），从左到右排列。
- 每个信号灯旁边显示一个字母键（Q, W, E…），从左到右排列。
- 标签需显著但不能遮挡下方设备状态。
- 非键盘模式下完全隐藏。

## 5. 游戏界面 — 列车列表（LeftPanel）

**承载"所有列车的宏观状态"**，用于快速定位异常列车与选择要操控的目标。

### 5.1 列表内容

合并显示两类列车，作为一个统一列表：
- **等待队列**：时刻表已生成、还未进站的列车
- **在站列车**：已经接入画布的列车

每行需展示：

| 信息 | 说明 |
|------|------|
| 车次号 | 如 G501 |
| 所在位置 | 文本化描述（"等待区" / "1 站台" / "进站线路" / "出站线路" / "线路中"） |
| 运行状态 | 状态徽章：等待进站 / 正在进站 / 停站中 / 可以出站 / 正在出站 / 运行中 / 停车 |
| 准点情况 | 准点 / 早点 N' / 晚点 N'（颜色编码见 §9） |
| 预计时间 | 预计进站时间 或 预计出站时间（停站中/进站中显示"预计 N 分钟后出站"） |
| 晚点扩散指示 | 三角符号 + 变化量（▲-1' 表示晚点减少、▼+2' 表示晚点加剧） |

### 5.2 交互

- **点击整行** → 选中该列车。选中态需在行上明确可见。
- **列表排序**：默认按"晚点恶化程度"降序（最紧急的列车在最上），让玩家优先处理高风险车次。
- **滚动**：列车数超出可视区时可滚动。

### 5.3 为空场景

- 开局尚未生成列车时，显示友好空态。
- 列车全部完成移交后，列表回归空态。

## 6. 游戏界面 — 时钟与控制（RightPanel）

### 6.1 游戏时钟

- 大字号显示虚拟游戏时间（HH:MM:SS）。
- 时间基于随机开局时刻（07:00–19:00），按 tick 推进。
- 需要醒目，是玩家判断准点的第一参照。

### 6.2 选中列车详情卡

选中一列车后显示，未选中则显示"未选择列车"空态。需呈现：

| 字段 | 说明 |
|------|------|
| 车型 | CR400AF / CR400BF / CRH380A |
| 车次号 | 作为最大标题 |
| 状态 | WAITING / moving / stopped |
| 位置 | 当前所在轨道 ID |
| 速度 | 带单位（km/h） |

### 6.3 调度指令按钮

**仅对已选中列车有效**，三个核心动作：

| 按钮 | 功能 | 允许时机 |
|------|------|----------|
| **允许进站（ADMIT）** | 将等待队列中的列车接入画布 | 列车在等待队列中 + 入口 300px 内无车 |
| **发车信号（DEPART）** | 向停站列车下达发车指令 | 列车停在站台 |
| **紧急停车（STOP）** | 立即使列车停下 | 列车处于运行中 |

按钮需有明显的**危险程度**区分（进站中性、发车积极、停车警示）。键盘模式下按钮上需显示快捷键提示（Tab / Shift+G / Shift+A）。

### 6.4 倍速控制

提供 5 档倍速：**暂停 / 1× / 2× / 5× / 10×**，当前档位需高亮。点击即切换。

---

## 7. 游戏界面 — 时刻表弹窗（ScheduleModal）

从游戏主界面可打开一个浮层，用于纵览时刻表与调度日志。**不阻塞游戏运行**，玩家可以在弹窗打开时继续操作倍速。

### 7.1 主体：时刻表

表格视图，显示**当前窗口内**（已等待 + 已接入 + 未来 30 分钟内将到达）的所有车次，按到达时间升序排列。

列定义：

| 列 | 说明 |
|---|---|
| 车次 | 如 G501 |
| 方向 | 上行 / 下行 |
| 进站时间 | 未到达时显示"预计：HH:MM:SS"，已进站显示实际时间 |
| 停站时间 | 格式如 `0'45`（分'秒） |
| 出站时间 | 逻辑同进站时间 |
| 准点情况 | 早点 / 准点 / 晚点 + 扩散指示（▲/▼ + 分钟） |

需要分页控件（上一页 / 下一页 / 当前页 / 总页数），因为行数可能很多。

### 7.2 弹窗侧边信息区

在弹窗内部再提供：

- **游戏时钟**（与主界面同步，弹窗打开时玩家仍需感知时间）
- **倍速控制**（箭头 + 当前倍速标签，供玩家一边看时刻表一边调速）
- **可切换的子标签页**（通过左右箭头切换）：
  - **调度日志**：时间戳 + 车次 + 事件（接入 / 发车 / 停车 / 进站停靠 / 移交完成）
  - **晚点扩散警告**：两组
    - ⚠ 晚点已加剧：列出晚点实际变差的列车及变化量
    - ⚠ 晚点即将加剧：列出接近扩散阈值（30 秒内将触发）的列车及剩余时间

### 7.3 关闭行为

- 点击遮罩层关闭。
- 鼠标移出弹窗主体时，右上角显现关闭按钮；移回则隐藏（这是现有的一个细节，设计师可自行调整更清晰的关闭方式）。

## 8. 全局交互模式

### 8.1 键盘 / 鼠标双模式

玩家可随时按 **Space** 键切换。切换时需弹出一个短时 toast 提示当前模式。

**鼠标模式**：所有道岔/信号灯直接点击操作。
**键盘模式**：画布上叠加键位标签，仅在此模式下响应按键。

完整键盘映射：

| 按键 | 功能 |
|------|------|
| `1–9, 0` | 切换道岔（从左到右） |
| `Q–P`（自动分配） | 切换信号灯（从左到右） |
| `Tab` | 允许进站（ADMIT） |
| `Shift+G` | 发车信号（DEPART） |
| `Shift+A` | 紧急停车（STOP） |
| `↑ / ↓` | 在列车列表中上/下选择 |
| `Shift+Space` | 暂停 / 继续 |
| `Shift+1/2/5/0` | 设倍速 1× / 2× / 5× / 10× |
| `Space` | 切换模式 |

UI 必须在相应按钮上展示键盘快捷键提示（当处于键盘模式时）。

### 8.2 Toast 提示

一种非阻塞的短时提示，用于以下场景：

- 模式切换成功
- 动作执行反馈（如"G501 允许进站"、"紧急停车"）
- 操作失败（如"入口拥堵！前车尚未驶离安全区"）
- 游戏失败提示（碰撞时）

需要两档视觉层级区分"模式提示"和"动作/错误提示"。

### 8.3 导航

游戏界面顶部需要两个常驻入口：

- **返回主菜单**（`← MENU`）：点击回到开始界面，重置全部游戏状态。
- **打开时刻表**：点击打开 §7 的弹窗。
- **当前车站名称**：作为顶部标题展示。

## 9. 信息编码惯例（语义色、状态文本）

不强制保留现有配色，但重构后需保持以下**语义**一致：

### 9.1 准点情况
- **早点**：积极意味（传统上用绿色）
- **准点**：中性（传统上用黄色）
- **晚点**：警示（传统上用红色）

### 9.2 列车状态
- **等待进站**：中性提醒色
- **正在进站 / 运行中**：进行态（传统蓝）
- **停站中**：需要玩家关注的停留态
- **可以出站**：可操作的积极态（传统绿）
- **正在出站**：已完成主要任务的状态
- **停车**：异常态（警示）
- **待命**：灰态

### 9.3 晚点扩散等级
- **improved（改善）**：▲ 符号 + 积极色
- **neutral（无变化）**：无图标
- **worsened（恶化）**：▼ 符号 + 警示色

### 9.4 信号灯
- **红灯**：禁止通行 + 发光效果
- **绿灯**：允许通行 + 发光效果
（发光是重要的功能性提示，不仅是装饰，便于在复杂画布中定位）

### 9.5 道岔激活出边
- 当前激活的出边路径需要以明显的高亮色叠加在轨道上，与普通轨道区分。

## 10. 响应式与适配

- 游戏主画布设计尺寸为 **2400 × 600** 的 SVG 坐标系，需在不同屏幕下等比适配。
- 时刻表弹窗设计基准尺寸 **2560 × 1432**（现实现为 JS 等比 scale），重构时可改用更灵活的响应式布局。
- 最小支持宽度约 **900px**，低于此需提示浏览器过窄或改为移动端布局。

---

## 11. 重构时可以重新审视的点

以下是现版 UI 中体验欠佳、建议重构时改进的方向：

1. **列车列表信息密度**：一行里塞了车次、位置、状态、准点、预计时间、扩散指示，信息过载；可以考虑分层/折叠。
2. **键盘标签叠加**：现在直接用大号红字覆盖在画布上，会遮挡线路；可以考虑更轻量的悬浮徽章。
3. **时刻表弹窗**：现使用 JS `transform: scale` 等比缩放固定尺寸，小屏幕下文字可能过小；建议真正响应式重构。
4. **模式切换反馈**：现用全局 toast，可以考虑在状态栏常驻当前模式指示器。
5. **选中列车的联动**：画布上选中列车的高亮目前较弱，选中后应与列表、详情卡三者联动更明显。
6. **暂停态的视觉传达**：目前仅靠倍速按钮高亮指示暂停，整个画布没有"已暂停"的全局视觉提示。
7. **新手引导**：目前无任何 onboarding，玩家很难理解"允许进站""发车""道岔切换"这些概念，建议设计引导流程。

## 12. 与代码交接的关键映射

重构 UI 时需要读/写的 Vue 组件：

| 组件 | 职责 |
|------|------|
| `src/App.vue` | 顶层状态、主循环、视图切换、键盘监听 |
| `src/components/StartScreen.vue` | 开始界面 |
| `src/components/GameCanvas.vue` | 主线路图 SVG |
| `src/components/panels/LeftPanel.vue` | 列车列表 |
| `src/components/panels/RightPanel.vue` | 时钟 / 详情卡 / 指令按钮 / 倍速 |
| `src/components/ScheduleModal.vue` | 时刻表弹窗 |

数据契约定义在：
- `src/core/types.ts`（`ScheduleEntry` / `TrainAction` / `DispatchLogEntry` / `DelaySpread` 等）
- `src/core/RailGraph.ts`（`RailMap` / `RailNode` / `RailEdge` / `TrainPhysics`）
- `src/core/constants.ts`（倍速选项、扩散阈值、停站时长等常量）

**重构建议**：UI 层可以完全重写，但保持对这些类型契约的消费不变即可与现有物理引擎、调度管理器无缝对接。
